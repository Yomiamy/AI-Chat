---

## 1. 功能背景與目的（Why）

### 問題陳述

目前 AI-Chat App 僅提供基本的對話功能，使用者無法：
- 在大量對話紀錄中快速找到特定內容
- 將對話紀錄匯出保存或分享給他人
- 篩選特定時間範圍或角色的訊息

### 解決方案價值

| 痛點 | 解決方案 | 使用者價值 |
|------|----------|------------|
| 對話紀錄難以查找 | 全文搜尋功能 | 快速定位重要資訊，節省時間 |
| 無法備份對話 | 匯出功能 | 資料可攜、知識保存 |
| 資訊回顧困難 | 搜尋 + 時間篩選 | 追溯過往討論脈絡 |

### 目標

1. 提供即時全文搜尋，回應時間 < 200ms
2. 支援多種匯出格式，滿足不同使用場景
3. 保持現有 UX 一致性，無學習成本

## 2. 使用者故事（User Stories）

### US-1：全文搜尋對話內容
> **作為** 使用者  
> **我想要** 輸入關鍵字搜尋所有對話紀錄  
> **以便** 快速找到之前討論過的特定主題或資訊

**補充說明**：
- 搜尋範圍包含使用者輸入與 AI 回覆
- 支援部分匹配（輸入「flutter」可匹配「Flutter」）
- 搜尋結果以高亮方式顯示匹配關鍵字

### US-2：匯出對話紀錄
> **作為** 使用者  
> **我想要** 將對話紀錄匯出為檔案  
> **以便** 備份保存或分享給他人

**補充說明**：
- 支援匯出全部對話或搜尋結果
- 可選擇不同格式（純文字 / Markdown）

## 3. 驗收條件（Acceptance Criteria）

### AC-1：搜尋功能

| 條件 | 驗收標準 |
|------|----------|
| 搜尋入口 | AppBar 顯示搜尋 icon，點擊後展開搜尋欄 |
| 即時搜尋 | 輸入後 300ms debounce 觸發搜尋 |
| 效能要求 | 100 筆訊息內，搜尋結果 < 200ms |
| 關鍵字高亮 | 搜尋結果中匹配文字以不同顏色標示 |
| 空結果處理 | 無匹配時顯示「找不到相關對話」提示 |
| 大小寫 | 搜尋不區分大小寫 |

### AC-2：匯出功能

| 條件 | 驗收標準 |
|------|----------|
| 匯出入口 | 透過 Menu（more_vert）進入 |
| 格式選項 | 支援 .txt 純文字與 .md Markdown |
| 匯出範圍 | 可選擇「全部對話」或「目前搜尋結果」 |
| 檔案命名 | 預設格式：`chat_export_YYYYMMDD_HHmmss.{ext}` |
| 分享整合 | 使用系統 Share Sheet，支援各平台原生分享 |

```
│ │ 請問 **Flutter** 的狀態管理... │ │  ← 關鍵字高亮

└────────────────────────────────────┘
```

### 5.2 匯出流程

```
[Menu] → [匯出對話] → [選擇格式 Dialog] → [系統 Share Sheet]
                           │
                           ├─ 純文字 (.txt)
                           └─ Markdown (.md)
```

### 5.3 匯出格式範例

**純文字格式（.txt）**：
```
AI Chat 對話紀錄
匯出時間：2026-05-04 15:30:00
==================

[你] 2026-05-03 14:32
請問 Flutter 的狀態管理怎麼選擇？

[AI] 2026-05-03 14:32
Flutter 常見的狀態管理方案有以下幾種...
```

**Markdown 格式（.md）**：
```markdown
# AI Chat 對話紀錄

> 匯出時間：2026-05-04 15:30:00

---

**你** *2026-05-03 14:32*

請問 Flutter 的狀態管理怎麼選擇？

---

**AI** *2026-05-03 14:32*

Flutter 常見的狀態管理方案有以下幾種...
```

## 6. 技術備註

### 資料來源
- 搜尋基於現有 `ChatMessage` Entity（ObjectBox）
- 利用 `ChatMessage_.content.contains()` 進行查詢

### 狀態管理
- 新增 `SearchEvent` / `SearchState` 擴展現有 BLoC
- 或獨立 `SearchBloc` 處理搜尋邏輯

### 匯出實作
- 使用 `share_plus` 或 `path_provider` + 系統分享
- 檔案暫存於 `getTemporaryDirectory()`
